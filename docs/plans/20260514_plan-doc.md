### D1. Components

```
┌──────────────────┐
│   claude.ai      │

         │ MCP over HTTPS (OAuth)
┌────────▼──────────────────────────────┐

│  ├─────────────────────────────────┤  │
│  │ NEW: invoice_risk_scan          │  │
│  │   ├─ Zod schema validate        │  │
│  │   ├─ SPARQL template render     │  │
│  │   └─ fetch(TUNNEL_URL)          │  │
│  └─────────────────────────────────┘  │
└─────────────────┬─────────────────────┘
                  │ HTTPS (Cloudflare Tunnel)
┌─────────────────▼─────────────────────┐
│  cloudflared (on jichu PC)            │
│  Tunnel: hvdc-bridge                  │
└─────────────────┬─────────────────────┘
                  │ HTTP localhost
┌─────────────────▼─────────────────────┐
│  Apache Jena Fuseki :3030             │
│  Dataset: /hvdc                       │
│  Graphs: OFCO, DSV, PKGS, PAY, META   │
└───────────────────────────────────────┘
```

### D2. Data Flow

1. claude.ai → Worker MCP `/mcp` POST
2. Worker → Zod validate input
3. Worker → SPARQL template + params
4. Worker → fetch `https://hvdc-bridge.{tunnel}.cloudflareaccess.com/sparql`
5. Tunnel → forward to localhost:3030
6. Fuseki → SPARQL execute → JSON 결과
7. Worker → parse → risk card structuredContent
8. Worker → MCP response → claude.ai render

### D3. Trust Boundaries

| Boundary | 입력 검증 | 출력 검증 |
|----------|---------|----------|
| claude.ai → Worker | OAuth Bearer + MCP scope | n/a |
| Worker handler | Zod strict | structuredContent schema |
| Worker → Tunnel | TUNNEL_TOKEN env | response Content-Type 확인 |
| Tunnel → Fuseki | localhost only | SPARQL response size limit (1MB) |

## E. Data Model & API Contract

### E1. Entity (SPARQL Mapping)

| Entity | Source Graph | Key Fields |
|--------|-------------|-----------|
| Invoice | `samsung:graph:OFCO`, `samsung:graph:DSV` | `ex:invoiceNumber`, `ex:totalAmount`, `ex:vatAmount`, `ex:dutyAmount`, `ex:invoiceDate` |
| Case | `samsung:graph:META` | `ex:caseNumber`, `ex:caseCode` |
| Supplier | `samsung:graph:OFCO` | `ex:supplierName` |

### E2. API Contract

#### `invoice_risk_scan` MCP Tool

**Input Schema** (Zod):
```typescript
z.object({
  supplier: z.enum(["OFCO", "DSV", "MAMMOET", "ALL"]).default("ALL"),
  date_from: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).optional(),
  date_to: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).optional(),
  min_amount_usd: z.number().nonnegative().optional(),
  limit: z.number().int().min(1).max(50).default(20),
})
```

**Output Schema** (structuredContent):
```typescript
{
  total_count: number,
  invoices: Array<{
    invoice_id: string,
    invoice_date: string | null,
    supplier: string,
    total_amount: number | null,
    currency: string,
    case_number: string | null,
    risk_types: Array<"MISSING_DATE"|"NEGATIVE_AMOUNT"|"MISSING_VAT"|"MISSING_DUTY"|"ZERO_TAX_DUTY">,
    severity: "HIGH" | "WARN" | "INFO",
    evidence_ref: { source_graph: string, source_file?: string }
  }>,
  query_meta: {
    sparql_template: "queries/03-invoice-risk-analysis.rq",
    executed_at: string,
    duration_ms: number,
    tunnel_health: "ok" | "degraded"
  },
  privacy: { piiMasked: true },
  actionBoundary: { readOnly: true, mutationAllowed: false }
}
```

**Error Responses**:
| Code | Trigger | UX Card |
|------|---------|--------|
| `VALIDATION_ERROR` | Zod fail | C-04 |
| `TUNNEL_UNAVAILABLE` | fetch timeout/5xx | C-03 |
| `NO_EVIDENCE` | empty SPARQL result | C-02 |
| `SPARQL_ERROR` | Fuseki 4xx/5xx | C-03 + log |

### E3. AuthN/AuthZ

- **claude.ai → Worker**: 기존 OAuth Bearer 재사용 (변경 ✗)
- **Worker → Tunnel**: `TUNNEL_TOKEN` (env var, wrangler secret)
- **Worker MCP scope**: `read:invoice_risk` 신규 추가 (write/upload scope 분리 유지)

|-----------|-----------|---------|

| Hop | Timeout |
|-----|---------|
